<h3>1. What is a file?</h3>
<p><strong>Answer:</strong><br />
A file is a named location which stores data or information permanently. A file is always stored inside a storage device using file name (e.g. STUDENT.MARKS). A file name normally has primary and secondary name separated by a “.”(DOT).</p>
<h3>2.What is a class?</h3>
<p><strong>Answer:</strong><br />
A class is a blueprint from which objects are created. A class contains methods and variables associated with an instance of a class.</p>
<h3>3.What is an object?</h3>
<p><strong>Answer:</strong><br />
An object is an instance of a class. For example<br />
class Abc{ &#8212;&#8211; This is a class<br />
int a; &#8212;&#8212; This is a variable<br />
public Abc(); &#8212;- This is contractor<br />
public static void main (String args[]) &#8212;&#8212;- This is a method<br />
{<br />
Abc a= new Abc(); &#8212;&#8212; This is object creation where ‘a’ is the reference variable or object name<br />
}<br />
}</p>

<p><strong>5.What is the different OOPS principle?</strong></p>
<p><strong>Answer:</strong><br />
The basic OOPS principle are as follows,</p>
<ul>
<li>Encapsulation</li>
<li>Abstraction</li>
<li>Inheritance</li>
<li>Polymorphism</li>
</ul>
<h3>6.What is inheritance?</h3>
<p><strong>Answer:</strong><br />
Inheritance is property in which the property of a parent class(Superclass) is passed on to child class(Subclass). For example<br />
class Abc{ &#8212;&#8211; This is a class<br />
int a; &#8212;&#8212; This is a variable<br />
public void abc(){} &#8212; Methods<br />
}<br />
class Xyz extends Abc &#8212;&#8211;(Extend is the keyword, Xyz is the subclass which inherits the properties of ABC parent class.)<br />
{<br />
public static void main (String args[]) &#8212;&#8212;- This is a method<br />
{<br />
Abc a= new Abc(); &#8212;&#8212; This is object creation where ‘a’ is the reference variable or object name<br />
}<br />
}</p>
<h3>7.What is polymorphism?</h3>
<p><strong>Answer:</strong><br />
Polymorphism is the ability of an object to take on multiple forms. Most commonly polymorphism is used in OOP when a parent class reference is used to refer to a child class object.</p>
<h3>8. What are the instance and class variables?</h3>
<p><strong>Answer:</strong><br />
Instance variable belongs to a particular instance of that class whereas Class variable. A class variable is also known as static variables. For example<br />
public class Abc{<br />
public int a; …….. This is an instance variable<br />
public static int a1;…….. This is a static or class variable<br />
……………………..<br />
……………..<br />
}</p>
<h3>9.Compare method and constructor?</h3>
<p><strong>Answer:</strong><br />
Constructor: Used to initialize the instance of a class.<br />
Method: Used to perform some function or operation.</p>
<p>Constructor: Doesn’t have a return type.<br />
Method: Has a return type.</p>
<h3>10. What is a singleton class?</h3>
<p><strong>Answer:</strong><br />
Singleton class limits the number of objects created for a class to one but gives the flexibility of creating more objects if the situation changes.</p>
<h3>11. What are the steps for creating the object?</h3>
<p><strong>Answer:</strong><br />
An object is first declared then instantiated and at last declared. For example<br />
Abc a= new Abc();</p>
<h3>12. What is the different type of access modifiers?</h3>
<p><strong>Answer:</strong><br />
There are four type of access modifiers as given below:-<br />
• Visible to the overall package. No modifier needed.<br />
• Private &#8211; Visible to class only.<br />
• Public – Visible to the world.<br />
• Protected – Visible to package and subclass.</p>
<h3>13.Which is the highest operator precedence in Java</h3>
<p><strong>Answer:</strong><br />
The operator with the highest preference is Postfix operators i.e () [].</p>
<h3>14.What is an array?</h3>
<p><strong>Answer:</strong><br />
The array is a container which holds the fixed number of similar data types.</p>
<h3>15. What is the difference between equals() and method and == operator?</h3>
<p><strong>Answer:</strong><br />
The equals() is a method and it matches the content of the strings whereas == is an operator and matches object or reference of the strings.</p>
<h3>16. Is string class final?</h3>
<p><strong>Answer:</strong><br />
Yes</p>

<h3>18.Difference between overloading and overriding?</h3>
<p><strong>Answer:</strong><br />
Overloading is when two or more methods in the same class have the same method name but different parameters(i.e different method signatures).<br />
Overriding is when two methods having the same method name and parameters (i.e., method signature) but one of the methods is in the parent class and the other is in the child class.</p>
<h3>19. What are multiple inheritances in Java?</h3>
<p><strong>Answer:</strong><br />
<a href="https://www.educba.com/course/java/">Java</a> supports multiple inheritances i.e the ability of a class to implement more than one interface. A class can implement multiple Interfaces but cannot extends multiple classes.</p>
<h3>20. What is a stream?</h3>
<p><strong>Answer:</strong><br />
A stream can be defined as the sequence of data. There is two type of streams.<br />
InputStream: Used to read data from a source.<br />
OutPut Stream: Used to write data into a destination.</p>
<h3>21. What is a Character stream?</h3>
<p><strong>Answer:</strong><br />
Java Character stream is basically used to perform input and output for 16 bit Unicode. The main classes users are FileReader and FileWriter which internally uses FileInputStream and FileOutputStream so the basic difference is that FileReader and FileWriter read and writes two bites at a time respectively.</p>
<h3>22. What is a Byte stream?</h3>
<p><strong>Answer:</strong><br />
Java Byte stream is basically used to perform input and output for 8 bit Unicode.<br />
The main classes related to byte streams are FileInputStream and FileOutputStream.</p>
<h3>23. What is an Interface?</h3>
<p><strong>Answer:</strong><br />
The interface is a reference type in <a href="https://www.educba.com/course/design-patterns-java/">Java</a>, similar to the class but its collection of abstract methods. A class can implement multiple interfaces.</p>
<h3>24.Difference between class and interface?</h3>
<p><strong>Answer:</strong><br />
Below are the difference between Interface and class:-</p>
<ul>
<li>The interface cannot be instantiated.</li>
<li>An interface doesn’t have any constructors.</li>
<li>Interface only have abstract methods.</li>
<li>A class implements an interface and extends a class.</li>
<li>An interface can extend multiple interfaces.</li>
</ul>
<h3>25. What is an abstract class?</h3>
<p><strong>Answer:</strong><br />
A class which contains the abstract keyword in a declaration is called abstract class. The properties of the abstract class are as follows:-</p>
<ul>
<li>Abstract classes may or may not contain <em>abstract methods</em> but, if a class has at least one abstract method, then it must be declared abstract.</li>
<li>The abstract class cannot be instantiated.</li>
<li>To use an abstract class, we have to inherit it from another class.</li>
<li>If we inherit an abstract class, then we have to provide implementations to all the abstract methods in it.</li>
</ul>
